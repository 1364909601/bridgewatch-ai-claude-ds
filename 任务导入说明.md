## 2. 本次采用的默认假设

- `Jira` 采用 `Task + Sub-task` 的一层层级结构
- `禅道` 采用 `任务导入` 的扁平结构，不做父子层级
- 不预填负责人
- 不预填实际开始日期和截止日期，避免错误日期进入系统
- 优先级映射如下：
  - `P0 -> Highest / 1`
  - `P1 -> High / 2`
  - `P2 -> Medium / 3`

## 3. Jira 文件说明

### 3.1 结构

- 顶层 5 条主线任务：
  - 数据主线
  - 算法主线
  - 平台主线
  - 展示主线
  - 测试与交付主线
- 其下为实际研发任务，作为 `Sub-task` 导入

### 3.2 导入建议

- 在 Jira 中使用 `CSV import`
- 确保目标项目已启用 `Task` 和 `Sub-task` 两种 Issue Type
- 导入时重点映射以下列：
  - `Issue Id`
  - `Summary`
  - `Issue Type`
  - `Parent`
  - `Priority`
  - `Description`

## 4. 禅道文件说明

### 4.1 结构

- 禅道文件按 `任务` 扁平化输出
- 主线信息已经放入 `任务名称` 中
- 前置依赖、计划周次、交付结果写入 `任务描述`

### 4.2 导入建议

- 文件列名按禅道任务导入常见字段整理：
  - `任务类型`
  - `任务名称`
  - `所属模块`
  - `相关需求`
  - `优先级`
  - `初预计`
  - `预计开始`
  - `截止日期`
  - `任务描述`
- 如果你的禅道版本要求先导出 Excel 模板，再导入：
  - 先从禅道导出任务导入模板
  - 将本文件列内容复制到模板同名列中
  - 再导入模板
