# 补充一：成功标准量化指标（补充 PRD §8）

## 1. 一期上线能力量化指标

### 1.1 普通桥梁四类风险识别效果

| 指标 | 目标值 | 说明 |
|---|---|---|
| **准确率（Precision）** | ≥ 80% | 正确识别事件数 / 系统告警总数 |
| **召回率（Recall）** | ≥ 75% | 正确识别事件数 / 实际事件总数 |
| **误报率（False Positive Rate）** | ≤ 20% | 误报数 / 系统告警总数 |
| **漏报率（False Negative Rate）** | ≤ 25% | 漏报数 / 实际事件总数 |
| **F1 Score** | ≥ 0.77 | 2 × Precision × Recall / (Precision + Recall) |

### 1.2 分场景识别效果基线

| 场景 | 准确率目标 | 召回率目标 | 说明 |
|---|---|---|---|
| 白天正常 | ≥ 85% | ≥ 80% | 最佳光照条件 |
| 夜间 | ≥ 70% | ≥ 65% | 低照度场景 |
| 雨雾 | ≥ 65% | ≥ 60% | 能见度受限场景 |

> 注：一期为研究验证级，以上目标为基线要求。如实际数据质量不足，可在项目启动后第2周根据数据盘点结果协商调整，但需书面确认。

### 1.3 事件级指标

| 指标 | 目标值 | 说明 |
|---|---|---|
| 事件生成延迟 | ≤ 30秒 | 从视频帧识别到事件入库的时间 |
| 多帧确认窗口 | 5-15帧可配 | 连续确认帧数范围 |
| 事件去重率 | ≥ 90% | 同一实际事件不重复生成告警的比例 |
| 事件闭环率 | 100% | 所有识别事件均可在事件中心查看、回放、复核 |

### 1.4 系统性能指标

| 指标 | 目标值 | 说明 |
|---|---|---|
| 视频导入吞吐 | ≥ 10路/小时 | 单节点导入处理能力 |
| 推理吞吐 | ≥ 5 FPS | 单GPU推理帧率 |
| 页面首屏加载 | ≤ 3秒 | 网页大屏首屏渲染时间 |
| 接口响应时间 | ≤ 500ms (P95) | 非推理类查询接口 |
| 系统可用性 | ≥ 95% | 工作时间（8:00-20:00）内系统可用率 |
| 并发用户 | ≥ 10 | 同时在线查看用户数 |

## 2. 原型验证量化指标

### 2.1 长大桥梁船撞原型

| 指标 | 目标值 | 说明 |
|---|---|---|
| 船只检测率 | ≥ 70% | 样例视频中船只检出比例 |
| 轨迹连续性 | ≥ 80% | 轨迹不中断帧数 / 总跟踪帧数 |
| 监测数据接入 | ≥ 1类 | 至少接通位移/振动/应变/水位/AIS中的一类 |
| 融合链路闭环 | 是 | 视频识别→监测接入→融合评分→页面展示全链路跑通 |

### 2.2 隧道融合预警原型

| 指标 | 目标值 | 说明 |
|---|---|---|
| 异常停车检测率 | ≥ 65% | 样例视频中异常停车检出比例 |
| 拥堵识别检测率 | ≥ 65% | 样例视频中拥堵场景检出比例 |
| 烟火识别检测率 | ≥ 70% | 样例视频中烟火检出比例 |
| 监测数据接入 | ≥ 1类 | 至少接通一类隧道监测数据 |
| 融合链路闭环 | 是 | 视频识别→监测接入→融合评分→页面展示全链路跑通 |

## 3. 验收测试方案

### 3.1 测试数据要求

- 每类事件至少准备 **20个正样本 + 10个负样本**
- 白天/夜间/雨雾三场景各覆盖 **30%以上** 样本
- 测试集与训练集 **严格分离**，测试集不参与模型训练

### 3.2 评估方法

- 采用 **逐帧评估 + 事件级评估** 双重标准
- 事件级评估：以事件时间窗口 IoU ≥ 0.5 且类型匹配为正确
- 由测试人员与算法工程师 **独立双评**，结果不一致时人工仲裁

### 3.3 报告要求

- 输出《一期识别效果评估报告》，含混淆矩阵、PR曲线、分场景统计
- 输出《原型验证报告》，含链路截图、样例视频、融合结果

## 4. 指标调整机制

- 项目启动后第2周（数据盘点完成后），如发现数据质量与预期偏差较大，可由项目负责人发起指标调整
- 调整需书面记录，明确调整原因、新指标值、影响范围
- 调整后指标不得低于基线的 **80%**
