# 补充四：算法技术方案详细设计（补充 PRD §11）

## 1. 总体算法架构

```
视频输入 → 预处理 → 目标检测/语义分割 → 时序分析 → 事件判定 → 事件输出
                                                    ↑
                                            监测数据融合（原型）
```

## 2. 普通桥梁识别算法详细方案

### 2.1 桥梁坍塌识别

**技术路线：关键区域检测 + 时序变化分析**

| 阶段 | 方法 | 说明 |
|---|---|---|
| 区域分割 | 语义分割模型（如 DeepLabV3+/SegFormer） | 对桥体结构区域进行像素级分割 |
| 变化检测 | 帧差法 + 结构相似度（SSIM） | 计算相邻帧/关键帧之间的结构变化 |
| 坍塌判定 | 规则引擎：结构缺失面积 > 阈值 + 持续帧数 > 阈值 | 多帧确认降低误判 |

**关键参数：**
- 结构缺失面积阈值：初始值 15%（可配置）
- 确认帧数：连续 5 帧以上触发事件
- 时间窗口：滑动窗口 30 秒

**模型选型建议：**
- 优先使用轻量级分割模型（如 BiSeNet），保证推理速度
- 预训练权重可基于 ImageNet/Cityscapes 迁移
- 微调数据：桥梁结构标注样本

### 2.2 桥面大变形识别

**技术路线：桥面区域定位 + 几何异常检测**

| 阶段 | 方法 | 说明 |
|---|---|---|
| 桥面定位 | 目标检测 + 区域提取 | 识别桥面区域并提取ROI |
| 变形检测 | 轮廓分析 + 曲率异常检测 | 分析桥面边缘线的曲率变化 |
| 变形判定 | 曲率突变幅度 > 阈值 + 多帧稳定确认 | 排除抖动和光照干扰 |

**关键参数：**
- 曲率突变阈值：初始值 20%（可配置）
- 稳定确认帧数：连续 8 帧
- 预处理：高斯平滑去噪

### 2.3 车辆积压识别

**技术路线：车辆检测 + 跟踪 + 密度速度估计**

| 阶段 | 方法 | 说明 |
|---|---|---|
| 车辆检测 | YOLOv8/YOLOv9 目标检测 | 检测视频中的车辆目标 |
| 车辆跟踪 | DeepSORT / ByteTrack | 对检测到的车辆进行多目标跟踪 |
| 密度估计 | 区域内车辆计数 / 区域面积 | 计算车辆空间密度 |
| 速度估计 | 轨迹位移 / 时间差 | 估算车流平均速度 |
| 积压判定 | 密度 > 阈值 AND 速度 < 阈值 AND 持续时间 > 阈值 | 三重条件联合判定 |

**关键参数：**
- 密度阈值：初始值 0.3 辆/平方米（可配置）
- 速度阈值：初始值 5 km/h（可配置）
- 持续时间阈值：60 秒
- 检测置信度阈值：0.5

### 2.4 桥面火灾识别

**技术路线：火焰检测 + 烟雾检测 + 多帧确认**

| 阶段 | 方法 | 说明 |
|---|---|---|
| 火焰检测 | 颜色空间分析（HSV/YCrCb） + CNN分类 | 先用颜色筛选候选区域，再用CNN确认 |
| 烟雾检测 | 运动检测 + 纹理分析 + CNN分类 | 检测烟雾的运动模式和纹理特征 |
| 联合判定 | 火焰 OR 烟雾 + 多帧确认 | 任一触发且多帧确认后生成事件 |

**关键参数：**
- 火焰颜色阈值：H∈[0,30], S∈[50,255], V∈[100,255]
- 烟雾运动阈值：光流幅值 > 2
- 确认帧数：连续 5 帧
- 反光过滤：排除水面反光（通过区域遮罩）

### 2.5 多帧确认与事件去重

**策略：滑动窗口 + 事件合并**

```
单帧识别结果 → 滑动窗口聚合 → 事件生成 → 相邻事件合并 → 最终事件
```

| 步骤 | 规则 |
|---|---|
| 窗口聚合 | 30秒滑动窗口内同类事件计数 |
| 事件触发 | 窗口内确认帧数 ≥ 阈值时生成事件 |
| 事件合并 | 相邻事件时间间隔 < 60秒时合并为同一事件 |
| 去重输出 | 合并后输出事件起止时间、关键帧、风险等级 |

## 3. 长大桥梁船撞融合算法

### 3.1 视频侧算法

| 能力 | 方法 | 输出 |
|---|---|---|
| 船只检测 | YOLOv8 水上目标检测 | 船只边界框、置信度 |
| 轨迹跟踪 | DeepSORT | 船只ID、轨迹点序列 |
| 桥区关系 | 轨迹点与桥墩区域空间关系计算 | 是否进入风险区域、最近距离 |

### 3.2 监测侧数据处理

| 数据类型 | 预处理 | 特征提取 |
|---|---|---|
| 位移 | 去噪、归一化 | 峰值、变化率 |
| 振动 | 滤波、频域分析 | 加速度RMS、频率主峰 |
| 应变 | 去漂移、归一化 | 应变增量、超限率 |
| 水位 | 时间对齐、插值 | 水位变化率 |
| AIS | 解析、坐标转换 | 船速、航向、距离桥墩距离 |

### 3.3 融合规则引擎

**规则示例：**

```
IF 船只进入桥墩风险区域
   AND 桥墩位移数据 > 基线值 × 1.2
   AND 振动RMS > 阈值
THEN risk_score += 40

IF 船只接近速度 > 阈值
   AND AIS距离 < 安全距离
THEN risk_score += 30

IF 监测数据无异常
   AND 船只轨迹未进入禁入区域
THEN risk_score -= 20
```

**风险等级映射：**
- 0-30：低风险
- 31-60：中风险
- 61-100：高风险

## 4. 隧道融合算法

### 4.1 视频侧算法

| 能力 | 方法 | 输出 |
|---|---|---|
| 异常停车 | 车辆检测 + 静止时间判断 | 静止超时车辆位置、时间 |
| 拥堵识别 | 车辆密度 + 速度估计 | 拥堵区域、拥堵程度 |
| 烟火识别 | 同桥梁火灾识别方案 | 烟火位置、置信度 |

### 4.2 监测侧数据处理

| 数据类型 | 用途 |
|---|---|
| CO浓度 | 辅助火灾/异常判断 |
| 能见度 | 辅助烟雾判断 |
| 温度 | 辅助火灾判断 |
| 风速 | 辅助烟雾扩散判断 |
| 车流量 | 辅助拥堵判断 |

### 4.3 融合规则

```
IF 视频检测到烟火
   AND (CO浓度异常 OR 温度异常)
THEN risk_level = high

IF 视频检测到拥堵
   AND 车流量数据确认
THEN risk_level = medium

IF 视频检测到异常停车
   AND 持续时间 > 阈值
THEN risk_level = medium
```

## 5. 模型训练策略

### 5.1 数据增强

| 增强方式 | 适用场景 | 参数 |
|---|---|---|
| 随机裁剪 | 全部 | 比例 0.5-1.0 |
| 随机翻转 | 全部 | 水平翻转概率 0.5 |
| 亮度调整 | 夜间场景 | 范围 0.6-1.4 |
| 模糊增强 | 雨雾场景 | 高斯核 3-7 |
| 噪声注入 | 全部 | 高斯噪声 σ=10-30 |
| Mixup | 全部 | α=0.2 |

### 5.2 训练配置

| 参数 | 建议值 |
|---|---|
| 优化器 | AdamW |
| 学习率 | 1e-4（预训练层）/ 1e-3（新层） |
| Batch Size | 16（根据GPU显存调整） |
| Epochs | 50-100 |
| 早停策略 | 验证集 loss 连续 5 epoch 不下降 |
| 硬件 | NVIDIA GPU，显存 ≥ 16GB |

### 5.3 模型评估指标

| 指标 | 说明 |
|---|---|
| mAP@0.5 | 目标检测任务 |
| mIoU | 语义分割任务 |
| Precision / Recall / F1 | 事件级评估 |
| FPS | 推理速度 |
| 模型大小 | 部署约束 |

## 6. 推理流程

```
1. 视频切片（每30秒一段）
2. 抽帧（2-5 FPS）
3. 前处理（resize, normalize）
4. 模型推理（batch inference）
5. 后处理（NMS, 置信度过滤）
6. 时序聚合（多帧确认）
7. 事件生成（去重、合并）
8. 结果入库
```

**推理优化建议：**
- 使用 TensorRT / ONNX Runtime 加速
- Batch 推理提升 GPU 利用率
- 视频切片并行处理
- 结果异步写入数据库
