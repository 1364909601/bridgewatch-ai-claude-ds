# 补充五：告警通知机制（新增模块）

## 1. 告警策略

### 1.1 告警级别

| 级别 | 触发条件 | 通知方式 | 响应要求 |
|---|---|---|---|
| **紧急** | 高风险事件（坍塌、火灾） | 页面弹窗 + 声音提示 + 消息推送 | 立即响应 |
| **重要** | 中风险事件（大变形、车辆积压） | 页面弹窗 + 消息推送 | 15分钟内响应 |
| **一般** | 低风险事件、原型预警 | 页面通知 | 工作时间处理 |
| **系统** | 模型异常、推理失败、数据断流 | 页面通知 + 日志记录 | 排班处理 |

### 1.2 告警规则

| 规则 | 说明 |
|---|---|
| 即时告警 | 高风险事件生成后立即推送 |
| 聚合告警 | 同一对象30分钟内同类事件合并为一条告警 |
| 静默期 | 同一事件类型在10分钟内不重复告警 |
| 升级机制 | 同一对象1小时内出现3次以上中风险事件，升级为紧急告警 |

## 2. 通知渠道

### 2.1 一期支持

| 渠道 | 说明 | 优先级 |
|---|---|---|
| 页面内通知 | 系统页面弹窗、声音提示、通知列表 | P0 |
| 短信通知 | 高风险事件发送短信到值班手机 | P1 |

### 2.2 后续扩展

| 渠道 | 说明 | 优先级 |
|---|---|---|
| 邮件通知 | 事件摘要邮件 | P2 |
| 企业微信/钉钉 | 群机器人推送 | P2 |
| 语音电话 | 紧急事件语音告警 | P2 |

## 3. 告警数据结构

### 3.1 告警记录表（建议新增）

```sql
CREATE TABLE alert_record (
    alert_id        VARCHAR(64)     PRIMARY KEY,
    event_id        VARCHAR(64),
    object_id       VARCHAR(64)     NOT NULL,
    alert_level     VARCHAR(32)     NOT NULL CHECK (alert_level IN ('urgent', 'important', 'normal', 'system')),
    alert_type      VARCHAR(64)     NOT NULL,
    alert_title     VARCHAR(255)    NOT NULL,
    alert_content   TEXT,
    notify_channel  VARCHAR(64)     NOT NULL,
    notify_status   VARCHAR(32)     NOT NULL DEFAULT 'pending',
    notify_time     TIMESTAMPTZ,
    ack_time        TIMESTAMPTZ,
    ack_user        VARCHAR(64),
    created_time    TIMESTAMPTZ     NOT NULL DEFAULT NOW()
);

COMMENT ON TABLE alert_record IS '告警通知记录';
COMMENT ON COLUMN alert_record.notify_status IS '通知状态：pending-待通知, sent-已通知, acked-已确认, expired-已过期';

CREATE INDEX idx_alert_record_event ON alert_record(event_id);
CREATE INDEX idx_alert_record_level ON alert_record(alert_level);
CREATE INDEX idx_alert_record_status ON alert_record(notify_status);
CREATE INDEX idx_alert_record_created ON alert_record(created_time);
```

## 4. 告警配置

系统应支持以下可配置项：

| 配置项 | 默认值 | 说明 |
|---|---|---|
| alert.enabled | true | 是否开启告警 |
| alert.sound.enabled | true | 是否开启声音提示 |
| alert.sound.file | alert.wav | 提示音文件 |
| alert.aggregation.window_minutes | 30 | 聚合窗口时间（分钟） |
| alert.silence.window_minutes | 10 | 静默期时间（分钟） |
| alert.escalation.count | 3 | 升级阈值（次数） |
| alert.escalation.window_minutes | 60 | 升级窗口时间（分钟） |
| alert.sms.enabled | false | 是否开启短信通知 |
| alert.sms.phones | | 短信接收号码列表 |
