# 补充六：用户与权限管理（补充 PRD 缺失模块）

## 1. 设计原则

- 一期为研究验证级，不做复杂权限体系
- 仅支持基础的登录认证和角色区分
- 用户数据存储在本地数据库，不对接外部认证系统
- 后续可扩展为 LDAP/SSO 对接

## 2. 角色定义

| 角色 | 权限 | 说明 |
|---|---|---|
| **管理员** | 全部权限 | 系统配置、用户管理、模型管理、数据管理 |
| **分析员** | 查看+复核+导出 | 事件查看、复核、视频回放、数据导出 |
| **查看者** | 仅查看 | 事件查看、视频回放、大屏展示 |

## 4. 用户表

```sql
CREATE TABLE sys_user (
    user_id         VARCHAR(64)     PRIMARY KEY,
    username        VARCHAR(64)     NOT NULL UNIQUE,
    password_hash   VARCHAR(255)    NOT NULL,
    real_name       VARCHAR(64),
    phone           VARCHAR(32),
    role            VARCHAR(32)     NOT NULL DEFAULT 'viewer' CHECK (role IN ('admin', 'analyst', 'viewer')),
    status          VARCHAR(32)     NOT NULL DEFAULT 'active' CHECK (status IN ('active', 'disabled')),
    last_login_time TIMESTAMPTZ,
    created_time    TIMESTAMPTZ     NOT NULL DEFAULT NOW(),
    updated_time    TIMESTAMPTZ     NOT NULL DEFAULT NOW()
);

COMMENT ON TABLE sys_user IS '系统用户';

CREATE INDEX idx_sys_user_username ON sys_user(username);
CREATE INDEX idx_sys_user_role ON sys_user(role);
```

## 5. 登录接口

### 5.1 登录

**POST** `/api/auth/login`

请求体：
```json
{
  "username": "admin",
  "password": "******"
}
```

响应：
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "expires_in": 86400,
    "user": {
      "user_id": "USR-001",
      "username": "admin",
      "real_name": "管理员",
      "role": "admin"
    }
  }
}
```

### 5.2 获取当前用户信息

**GET** `/api/auth/me`

需要 Header: `Authorization: Bearer <token>`

### 5.3 修改密码

**POST** `/api/auth/change-password`

请求体：
```json
{
  "old_password": "******",
  "new_password": "******"
}
```

## 7. Token 策略

| 参数 | 值 |
|---|---|
| 签名算法 | HS256 |
| 有效期 | 24小时 |
| 续期 | 登录后自动续期 |
| 存储 | 前端 localStorage |
