## 2. 项目背景

当前桥梁与隧道风险识别主要依赖人工巡检、人工研判和分散式监测，存在以下问题：

- 无法快速从视频中识别坍塌、桥面大变形、车辆积压、火灾等重大风险
- 长大桥梁船撞风险需要结合视频与结构健康监测数据综合判断，人工分析成本高
- 隧道场景中的异常行为、烟火、拥堵等风险难以及时联动监测数据进行预警
- 监测中心缺少统一展示入口，无法形成事件查看、回放、统计、原型验证的完整闭环

本项目拟建设一套面向监测中心的“普通桥梁视频风险识别上线系统 + 长大桥梁船撞融合预警原型 + 隧道融合预警原型”，在 2 个月内完成一期可上线能力，并同步完成两类融合原型验证。

## 3. 产品目标

### 3.1 总体目标

- 建立普通桥梁重大风险的视频识别能力
- 建立长大桥梁船撞场景的视频与监测数据融合预警原型
- 建立隧道场景的视频与监测数据融合预警原型
- 为监测中心提供统一的大屏展示、事件查询、历史回放和统计分析能力

### 3.2 一期目标

- 上线普通桥梁视频风险识别系统
- 支持桥梁坍塌、桥面大变形、车辆积压、桥面火灾四类事件识别
- 支持视频管理、事件生成、事件查询、结果展示、历史回放
- 完成 Linux 私有化部署和网页大屏交付

### 3.3 二期原型目标

- 完成长大桥梁船撞融合预警原型
- 完成隧道融合预警原型
- 跑通数据接入、融合计算、风险输出、页面展示闭环

## 5. 使用场景

### 5.1 场景一：普通桥梁风险识别

监测中心导入或接收无人机采集视频后，系统自动完成视频预处理和模型识别，并输出以下事件：

- 桥梁坍塌
- 桥面大变形
- 车辆积压
- 桥面火灾

识别结果在网页大屏展示，用户可查看事件截图、事件片段、事件时间和所属桥梁。

### 5.2 场景二：长大桥梁船撞融合预警原型

系统接入长大桥梁视频数据和结构健康监测数据，对船只目标、航迹、桥墩区域关系进行识别，并结合监测数据变化进行风险评分，输出原型级风险结果，在专题页展示。

### 5.3 场景三：隧道融合预警原型

系统接入隧道视频和监测数据，对烟火、异常停车、拥堵等事件进行识别，并联合监测信号输出风险等级，在原型页展示。

## 8. 成功标准

当前项目尚未给出明确数值阈值，建议以以下方式定义成功标准：

- 普通桥梁四类风险识别能力可运行、可展示、可回放
- 在白天、夜间、雨雾场景下均可完成识别测试
- 可形成事件级结果，而非仅单帧结果
- 可对长大桥梁船撞场景完成视频与监测数据融合原型演示
- 可对隧道场景完成视频与监测数据融合原型演示

- 准确率

## 9. 总体方案

本项目采用“数据层 + 算法层 + 服务层 + 展示层”的分层方案，优先保证一期上线闭环，避免在研究验证阶段引入过重平台复杂度。

### 9.1 架构目标

- 支撑视频识别与多源融合
- 支撑模型快速迭代
- 支撑私有化交付
- 支撑网页大屏可视化

### 9.2 总体流程

1. 接入无人机视频和监测数据
2. 执行视频预处理和数据清洗
3. 执行视频识别与异常检测
4. 执行融合规则计算和风险评分
5. 生成事件结果和统计结果
6. 在网页大屏进行查询、回放和展示

## 11. 算法方案

### 11.1 普通桥梁识别算法

#### 11.1.1 桥梁坍塌

- 使用关键结构区域识别 + 时序变化分析
- 识别桥体大范围缺失、结构突变、整体形态异常

#### 11.1.2 桥面大变形

- 使用桥面区域识别 + 几何变化判断
- 通过多帧稳定分析降低误判

#### 11.1.3 车辆积压

- 使用车辆检测 + 跟踪 + 速度密度估计
- 判断车流速度下降、停滞时间增加和空间聚集

#### 11.1.4 桥面火灾

- 使用火焰与烟雾检测
- 引入多帧确认机制，降低反光和噪声误报

### 11.2 长大桥梁船撞融合算法

- 视频侧：船只检测、轨迹跟踪、桥区接近分析
- 监测侧：位移、振动、应变、水位、AIS 等数据接入
- 融合侧：时间同步、规则引擎、风险评分

### 11.3 隧道融合算法

- 视频侧：异常停车、拥堵、烟火识别
- 监测侧：环境监测、设备状态、结构监测数据接入
- 融合侧：联合触发与风险等级计算

## 13. 技术选型建议

| 层级 | 建议技术 |
|---|---|
| 算法训练/推理 | Python、PyTorch |
| 后端服务 | FastAPI |
| 前端展示 | React 19 + TypeScript + Vite、Recharts + Framer Motion、Tailwind CSS v4 |
| 数据存储 | PostgreSQL |
| 文件存储 | MinIO |
| 容器化 | Docker Compose |

> **注：** 前端技术栈已在实际 demo（bridgewatch-ai）中验证通过。Recharts 负责数据图表绘制，Framer Motion 负责页面转场动画，Tailwind CSS v4 负责样式体系。AI 摘要助手可选集成 `@google/genai` SDK，正式环境需改为后端代理调用。

选型原则如下：

- 以成熟开源技术为主，降低研究验证成本
- 以快速开发和快速部署为优先
- 保证后续可平滑演进

## 18. 风险与边界

### 18.1 主要风险

- 无人机视频机位变化大，影响识别稳定性
- 夜间和雨雾场景会降低识别效果
- 极端事件样本较少，影响模型泛化
- 融合效果依赖监测数据质量和时间同步能力
- 2 个月周期对功能范围有明显约束

### 18.2 边界说明

- 本期重点是研究验证和能力闭环，不是商用级平台建设
- 本期不承诺高并发、多区域、多租户能力
- 本期不承诺秒级硬实时告警

## 24. 页面原型说明

### 24.1 页面设计目标

- 满足监测中心统一查看、快速定位、事件回放、原型验证的使用需求
- 页面结构尽量简单，优先支撑一期上线和原型演示
- 页面信息分层清晰，先总览，再下钻到对象、事件和视频
- 页面交互以查询、筛选、查看、回放为主，不引入复杂操作流

### 24.2 页面结构总览

建议系统采用左侧菜单 + 顶部状态栏 + 主内容区的结构。

#### 24.2.1 一级菜单

- 首页总览
- 事件中心
- 视频回放
- 普通桥梁专题
- 长大桥梁船撞专题
- 隧道融合预警专题
- 模型运行看板

#### 24.2.2 公共顶部区域

所有页面顶部建议统一展示以下信息：

- 系统名称
- 当前时间
- 当前数据范围
- 当前选中对象
- 最近一次数据更新时间

#### 24.2.3 公共筛选条件

多个页面可复用以下筛选项：

- 对象类型：桥梁 / 隧道
- 对象名称
- 时间范围
- 场景类型：白天 / 夜间 / 雨雾
- 风险类型
- 风险等级

### 24.3 页面一：首页总览

#### 24.3.1 页面目标

- 让监测中心快速掌握当前系统覆盖对象、事件分布和风险态势
- 作为系统默认首页，承担总入口功能

#### 24.3.2 页面布局

- 顶部：系统标题区 + 全局筛选区
- 左侧：对象概览区
- 中间：风险总览卡片 + 趋势图
- 右侧：重点事件列表 + 模型运行状态
- 底部：专题入口区

#### 24.3.3 页面内容

| 区域 | 内容 |

| 顶部标题区 | 系统名称、当前时间、数据更新时间 |

## 27. 交互状态说明

### 27.1 通用状态

所有页面建议统一处理以下状态：

| 状态 | 触发条件 | 页面表现 |
|---|---|---|
| 初始状态 | 首次进入页面 | 展示默认时间范围和默认统计数据 |
| 加载中 | 请求接口中 | 显示骨架屏或加载动画 |
| 无数据 | 无符合条件数据 | 显示空状态提示和返回建议 |
| 加载失败 | 接口异常或网络异常 | 显示错误提示和重试按钮 |
| 权限不足 | 无访问权限 | 显示无权限提示页 |

### 27.2 查询交互规则

| 场景 | 规则 |
|---|---|
| 切换对象类型 | 清空对象名称下拉，并重新加载可选对象 |
| 修改时间范围 | 自动影响统计卡片、图表和列表结果 |
| 从总览跳到事件中心 | 自动带入当前筛选条件 |
| 从事件跳到回放页 | 自动定位事件时间点和对应视频 |
| 切换专题对象 | 自动刷新样例、统计和图表 |

### 27.3 事件复核交互规则

| 场景 | 规则 |
|---|---|
| 打开事件详情 | 默认展示只读信息和当前复核状态 |
| 修改复核状态 | 必须点击保存复核后生效 |
| 保存成功 | 列表和详情同步更新复核状态 |
| 保存失败 | 保留当前输入并提示重试 |

### 27.4 视频回放交互规则

| 场景 | 规则 |
|---|---|
| 点击事件回放 | 自动跳转并定位到事件时间 |
| 点击时间轴标记点 | 播放器跳转到对应事件位置 |
| 切换识别框开关 | 仅影响前端显示，不重新请求视频 |
| 导出片段 | 以当前事件时间窗口为默认导出范围 |
